1️⃣ Introduction to Software Development Types of Software • SDLC • Compilers
Topics:
  • Types of Software, SDLC phases, Programming Languages, Compilers & Interpreters
Q1: Define software and explain its main classifications.

Ans: Software is a set of programs and data that instruct a computer to perform tasks. It is mainly classified into system software (operating systems, device drivers), application software (word processors, browsers) and programming software (compilers, interpreters, IDEs). Each class serves different purposes: system software manages hardware, applications serve users, and programming tools aid development.

سوال 1: سافٹ ویئر کی تعریف کریں اور اس کی بنیادی درجہ بندی بیان کریں۔

جواب: سافٹ ویئر ہدایات اور ڈیٹا کا مجموعہ ہے جو کمپیوٹر کو کام کرنے کی رہنمائی دیتا ہے۔ اسے عام طور پر سسٹم سافٹ ویئر (آپریٹنگ سسٹم، ڈرائیور)، ایپلیکیشن سافٹ ویئر (ورڈ پروسیسر، براؤزر) اور پروگرامنگ سافٹ ویئر (کمپائلرز، انٹرپریٹر، IDE) میں تقسیم کیا جاتا ہے۔ ہر قسم کا مقصد مختلف ہوتا ہے۔

Q2: What is SDLC? Describe its main phases with purpose.

Ans: SDLC (Software Development Life Cycle) is a structured approach for building software. Phases: Requirement Analysis (gathering needs), Design (architecture & modules), Implementation (coding), Testing (finding/fixing bugs), Deployment (release), Maintenance (updates & fixes). Each phase ensures quality and control over development.

سوال 2: SDLC کیا ہے؟ اس کے بنیادی مراحل اور ہر مرحلے کا مقصد بیان کریں۔

جواب: SDLC ایک منظم طریقہ ہے جو سافٹ ویئر بنانے کے لیے اختیار کیا جاتا ہے۔ مراحل: ضروریات کی تحلیل، ڈیزائن، عمل آوری (کوڈنگ)، ٹیسٹنگ، تعیناتی، اور بحالی۔ ہر مرحلہ معیار اور کنٹرول کو یقینی بناتا ہے۔

Q3: Differentiate between compiler and interpreter with examples.

Ans: Compiler translates entire source code to machine code before execution (example: C/C++ compilers), producing an executable. Interpreter translates and executes code line by line (example: Python), which eases debugging but typically runs slower. Hybrid methods (bytecode + VM) exist like Java's JVM.

سوال 3: کمپائلر اور انٹرپریٹر میں فرق بتائیں مثال کے ساتھ۔

جواب: کمپائلر پورا سورس کو مشین کوڈ میں ایک ساتھ تبدیل کرتا ہے اور ایک executable بناتا ہے (مثال: C)، جبکہ انٹرپریٹر لائن بہ لائن کوڈ کو چلتا ہے (مثال: Python)، جس سے ڈیبگ آسان ہوتا ہے مگر رفتار عام طور پر کم ہوتی ہے۔

Q4: Explain characteristics of good software.

Ans: Good software should be correct, reliable, efficient, usable, maintainable and secure. Correctness ensures requirements are met; reliability avoids crashes; efficiency optimizes resources; maintainability allows updates; security protects data and users.

سوال 4: اچھے سافٹ ویئر کی خصوصیات بیان کریں۔

جواب: اچھا سافٹ ویئر درست، قابل اعتماد، موثر، استعمال میں آسان، قابلِ ترمیم اور محفوظ ہونا چاہیے۔ یہ خصوصیات صارف کی ضروریات کو پورا کرنے اور سسٹم کو طویل عرصے تک مستحکم رکھنے میں مدد دیتی ہیں۔

Q5: What is documentation and why is it important?

Ans: Documentation records requirements, design, code comments and user guides. It is crucial for maintenance, team communication, onboarding new developers and for users to understand how to operate the system.

سوال 5: دستاویزات کیا ہیں اور یہ کیوں ضروری ہیں؟

جواب: دستاویزات میں ضروریات، ڈیزائن، کوڈ کمنٹس اور یوزر گائیڈ شامل ہوتے ہیں۔ یہ بحالی، ٹیم کے درمیان رابطے اور نئے ڈویلپرز کی تربیت کے لیے ضروری ہیں۔

Q6: Define open-source and proprietary software with pros/cons.

Ans: Open-source software shares source code for public use/modification (e.g., Linux); pros: transparency, community support; cons: varying support. Proprietary software is closed-source and licensed (e.g., Windows); pros: vendor support; cons: license cost, less transparency.

سوال 6: اوپن سورس اور پروپرائٹری سافٹ ویئر کی تعریف اور فوائد/نقصانات بتائیں۔

جواب: اوپن سورس سافٹ ویئر کا سورس کوڈ عوام کے لیے دستیاب ہوتا ہے (مثال: Linux)، فوائد: شفافیت، کمیونٹی سپورٹ؛ نقصانات: سپورٹ کا فرق۔ پروپرائٹری سافٹ ویئر بند سورس اور لائسنسڈ ہوتا ہے (مثال: Windows)، فوائد: رسمی سپورٹ؛ نقصانات: لائسنس لاگت۔

Q7: Explain software lifecycle models: Waterfall and Agile.

Ans: Waterfall is linear sequential model with distinct phases; changes are hard once a phase completes. Agile is iterative and incremental with regular feedback, allowing flexible changes and faster delivery of working increments.

سوال 7: سافٹ ویئر لائف سائیکل ماڈلز Waterfall اور Agile کی وضاحت کریں۔

جواب: Waterfall ایک سلسلہ وار ماڈل ہے جہاں ہر مرحلہ مکمل ہونے کے بعد اگلے میں جائے گا؛ تبدیلی مشکل ہوتی ہے۔ Agile تکراری اور لچکدار ماڈل ہے جو بار بار فیڈبیک اور چھوٹے ریلیزز پر مشتمل ہوتا ہے۔

Q8: What is version control and name popular systems.

Ans: Version control tracks changes in code/documents, enabling collaboration and rollback. Popular systems: Git (distributed), SVN (centralized). GitHub/GitLab are hosting platforms for Git repositories.

سوال 8: ورژن کنٹرول کیا ہے اور مشہور سسٹمز کے نام بتائیں۔

جواب: ورژن کنٹرول کوڈ اور ڈیٹا میں تبدیلیوں کو ٹریک کرتا ہے، تعاون اور rollback ممکن بناتا ہے۔ مشہور سسٹمز: Git، SVN؛ GitHub/GitLab Git کے لیے ہوسٹنگ فراہم کرتے ہیں۔

Q9: Discuss role of testing and types of testing.

Ans: Testing ensures software meets requirements and is defect-free. Types: unit, integration, system, acceptance, regression, performance and security testing. Each type addresses different risk areas.

سوال 9: ٹیسٹنگ کا رول اور ٹیسٹنگ کی اقسام بتائیں۔

جواب: ٹیسٹنگ اس بات کی ضمانت دیتی ہے کہ سافٹ ویئر ضروریات پورا کرے اور خرابیاں کم ہوں۔ اقسام: یونٹ، انٹیگریشن، سسٹم، ایکسیپٹنس، ریگریشن، پرفارمنس، سیکیورٹی۔

Q10: What is maintainability and why it matters?

Ans: Maintainability is ease of updating software to fix bugs or add features. High maintainability reduces cost, speeds up updates, and improves reliability across the software lifecycle.

سوال 10: مینٹی نیبیلٹی کیا ہے اور یہ کیوں اہم ہے؟

جواب: مینٹی نیبیلٹی سے مراد سافٹ ویئر کو آسانی سے اپ ڈیٹ یا درست کرنا ہے۔ اچھی مینٹی نیبیلٹی لاگت کم کرتی ہے، اپ ڈیٹس تیز کرتی ہے اور نظام کی اعتمادیت بڑھاتی ہے۔

2️⃣ Python Programming Syntax • Data Types • Control Structures • I/O
Topics:
  • Python basics, variables, control, functions, file handling
Q1: Describe Python and its main features.

Ans: Python is a high-level interpreted language known for readability, dynamic typing and extensive libraries. Features include simple syntax, cross-platform support, automatic memory management and a large standard library suitable for web, data science and scripting.

سوال 1: Python کی وضاحت اور اہم خصوصیات بتائیں۔

جواب: Python ایک اعلی سطحی انٹرپریٹڈ زبان ہے جو پڑھنے میں آسان اور ڈائنامک ٹائپنگ فراہم کرتی ہے۔ خصوصیات: سادہ نحو، کراس پلیٹ فارم، خودکار میموری مینجمنٹ اور مضبوط اسٹینڈرڈ لائبریری۔

Q2: Write a Python program to add two numbers with input validation.

Ans: Example: try: a=float(input('A: ')); b=float(input('B: ')); print('Sum=',a+b) except: print('Enter valid numbers'). Use try/except to handle invalid inputs.

سوال 2: دو نمبروں کو جمع کرنے کا Python پروگرام لکھیں (ان پٹ ویلیڈیشن کے ساتھ).

جواب: مثال: try: a=float(input('A: ')); b=float(input('B: ')); print('Sum=',a+b) except: print('درست نمبرز درج کریں')۔

Q3: Explain Python data types with examples.

Ans: Main types: int, float, str, list, tuple, dict, set, bool. Example: int 5, float 3.14, str 'hello', list [1,2,3], dict {'a':1}.

سوال 3: Python ڈیٹا ٹائپس کی وضاحت مثال کے ساتھ کریں۔

جواب: اہم ٹائپس: int، float، str، list، tuple، dict، set، bool۔ مثال: int 5، float 3.14، str 'hello'۔

Q4: Discuss conditional statements and give example.

Ans: Conditional statements (if-elif-else) control flow based on boolean conditions. Example: if x>0: print('Positive') elif x==0: print('Zero') else: print('Negative').

سوال 4: کنڈیشنل اسٹیٹمنٹس پر بات کریں اور مثال دیں۔

جواب: if-elif-else بیان حالات کی بنیاد پر کوڈ کا راستہ منتخب کرتے ہیں۔ مثال: if x>0: print('Positive') …

Q5: Explain loops (for, while) with examples.

Ans: for iterates over sequences (for i in range(5):), while repeats until condition false. Use break/continue for control. Loops are used for iteration and repetition tasks.

سوال 5: لوپس (for, while) کی وضاحت کریں مثال کے ساتھ۔

جواب: for ترتیب پر چلتا ہے (for i in range(5):)، while تب تک چلے جب تک شرط سچ ہو۔ break/continue کنٹرول کے لیے ہیں۔

Q6: What are functions and how to define them?

Ans: Functions encapsulate reusable logic. Define with def: def add(a,b): return a+b. Functions enhance modularity and testing.

سوال 6: فنکشنز کیا ہیں اور انہیں کیسے ڈیفائن کیا جاتا ہے؟

جواب: فنکشنز دوبارہ استعمال کے قابل کوڈ بلاکس ہیں۔ def add(a,b): return a+b۔ یہ ماڈیولیریٹی بڑھاتے ہیں۔

Q7: Explain file handling basics in Python.

Ans: Use open(filename, mode). Modes: 'r','w','a','rb','wb'. Use with-statement: with open('f.txt','r') as f: data=f.read() to auto-close.

سوال 7: Python میں فائل ہینڈلنگ کی بنیادی باتیں بتائیں۔

جواب: open استعمال کریں، موڈز 'r','w','a' وغیرہ۔ باقاعدہ استعمال: with open('f.txt','r') as f: … جو آٹو کلوز بھی کرتا ہے۔

Q8: What is exception handling? Give example.

Ans: Use try-except to catch runtime errors. Example: try: x=1/0 except ZeroDivisionError: print('Cannot divide'). finally block runs for cleanup.

سوال 8: استثناء (Exceptions) کیا ہیں؟ مثال دیں۔

جواب: try-except بلاکس رن ٹائم ایررز کو ہینڈل کرتے ہیں۔ مثال اوپر دی گئی ہے۔

Q9: Compare list and tuple.

Ans: Lists are mutable (can change), tuples are immutable (fixed). Use lists for changeable collections and tuples for fixed records and faster access.

سوال 9: لسٹ اور ٹیوپل میں موازنہ کریں۔

جواب: لسٹ تبدیل ہو سکتی ہے؛ ٹیوپل مستقل ہوتی ہے۔ بدلتے ڈیٹا کے لیے لسٹ بہتر ہے۔

Q10: What are modules and how to use them?

Ans: Modules are .py files with functions/variables. Use import: import math or from math import sqrt. Packages organize modules into directories.

سوال 10: ماڈیولز کیا ہیں اور انہیں کیسے استعمال کیا جاتا ہے؟

جواب: ماڈیولز .py فائلیں ہیں جن میں فنکشنز ہوتے ہیں۔ import کمانڈ سے استعمال کریں۔ پیکیجز ماڈیولز کو ڈائریکٹری میں منظم کرتے ہیں۔

3️⃣ Algorithms & Problem Solving Definitions • Flowchart • Pseudocode • Complexity
Topics:
  • Algorithm properties, flowcharts, pseudocode, basic sorting/searching
Q1: Define algorithm and list its properties.

Ans: An algorithm is a finite step-by-step procedure to solve a problem. Properties: finiteness, definiteness, input, output, effectiveness and clarity. Good algorithms are correct and efficient.

سوال 1: الگورتھم کی تعریف کریں اور اس کی خصوصیات بتائیں۔

جواب: الگورتھم مسئلہ حل کرنے کا محدود مرحلہ وار طریقہ ہے۔ خصوصیات: محدودیت، واضح ہونا، ان پٹ اور آؤٹ پٹ، مؤثر ہونا وغیرہ۔

Q2: Explain pseudocode and write pseudocode for factorial.

Ans: Pseudocode is informal description of steps. Example: SET fact=1; FOR i=1 TO n DO fact = fact * i; END; PRINT fact.

سوال 2: پسوڈوکوڈ کیا ہے اور فیکٹوریل کا پسوڈوکوڈ لکھیں۔

جواب: پسوڈوکوڈ اقدامات کی غیر رسمی تفصیل ہے۔ مثال اوپر دی گئی ہے۔

Q3: Describe flowchart symbols and steps to find largest of two numbers.

Ans: Symbols: Oval=start/end, Parallelogram=I/O, Rectangle=process, Diamond=decision. Steps: Start→Input A,B→If A>B output A else output B→End.

سوال 3: فلوچارٹ سمبولز بتائیں اور دو نمبروں میں بڑا نمبر معلوم کرنے کے مراحل لکھیں۔

جواب: سمبولز اور مراحل اوپر بیان کیے گئے ہیں۔

Q4: Explain linear and binary search with complexities.

Ans: Linear search checks elements sequentially: O(n). Binary search on sorted array halves search space each step: O(log n). Binary is faster but requires sorted data.

سوال 4: لینیئر اور بائنری سرچ کی وضاحت کریں اور کمپلیکسٹی بتائیں۔

جواب: لینیئر O(n) ہے، بائنری O(log n) ہے مگر بائنری کے لیے ڈیٹا مرتب ہونا ضروری ہے۔

Q5: What is recursion? Give example.

Ans: Recursion is when a function calls itself with smaller input and a base case. Example factorial: fact(n)=n*fact(n-1), with fact(0)=1.

سوال 5: ری کرشن کیا ہے؟ مثال دیں۔

جواب: جب فنکشن خود کو چھوٹے مسئلے کے لیے کال کرے تو اسے ری کرشن کہتے ہیں۔ مثال فیکٹوریل۔

Q6: Define time complexity and Big-O notation.

Ans: Time complexity measures run time relative to input size; Big-O gives upper bound (e.g., O(n), O(n^2)). It helps compare algorithm efficiency.

سوال 6: ٹائم کمپلیکسٹی اور Big-O کی تعریف کریں۔

جواب: ٹائم کمپلیکسٹی ان پٹ سائز کے لحاظ سے پروگرام کے چلنے کے وقت کو ماپتی ہے؛ Big-O اوپری حد بتاتی ہے۔

Q7: Explain bubble sort and its complexity.

Ans: Bubble sort repeatedly swaps adjacent elements if out of order. Time complexity average/worst O(n^2); best O(n) if already sorted. Simple but inefficient for large arrays.

سوال 7: ببل سورٹ کی وضاحت کریں اور اس کا کمپلیکسٹی بتائیں۔

جواب: ببل سورٹ میں پڑوسی عناصر کو بدلتے ہیں جب ترتیب درست نہ ہو؛ اوسط/بدترین O(n^2) ہے۔

Q8: What is a data structure? Name common ones.

Ans: Data structure organizes data for efficient access. Common: arrays, linked lists, stacks, queues, trees, graphs, hash tables.

سوال 8: ڈیٹا اسٹریکچر کیا ہے؟ مشہور اقسام بتائیں۔

جواب: ڈیٹا اسٹریکچر ڈیٹا کو منظم کرتے ہیں؛ مثالیں اوپر دی گئی ہیں۔

Q9: How to test an algorithm?

Ans: Test with varied inputs including edge cases, verify outputs, analyze complexity and memory usage, use unit tests and compare with expected results to ensure correctness and performance.

سوال 9: الگورتھم کو کیسے ٹیسٹ کریں؟

جواب: مختلف ان پٹس اور ایج کیسز کے ساتھ ٹیسٹ کریں، آؤٹ پٹ کی جانچ کریں، یونٹ ٹیسٹنگ کریں اور کارکردگی کا تجزیہ کریں۔

Q10: Explain greedy approach with an example.

Ans: Greedy algorithms make locally optimal choices hoping for global optimum. Example: coin change (with canonical coin systems) or activity selection where choosing earliest finishing activity yields optimality.

سوال 10: گریڈی اپروچ کی وضاحت کریں اور مثال دیں۔

جواب: گریڈی ہر قدم پر بہترین مقامی فیصلہ لیتا ہے؛ مثال: سرگرمیوں کے انتخاب میں جلد ختم ہونے والی سرگرمی منتخب کرنا۔

4️⃣ Computational Structures Binary • Logic Gates • Boolean Algebra
Topics:
  • Binary/Others, logic gates, truth tables, Boolean simplification
Q1: Explain number systems and convert binary 1010 to decimal.

Ans: Number systems include binary (base2), octal, decimal, hex. 1010₂ = 1×2³ +0×2² +1×2¹ +0 = 8+0+2+0 = 10₁₀.

سوال 1: نمبر سسٹمز کی وضاحت کریں اور 1010₂ کو ڈیسمل میں بدلیں۔

جواب: اوپر وضاحت کے مطابق، 1010₂ = 10₁₀۔

Q2: Define AND, OR, NOT gates and give truth tables.

Ans: AND outputs 1 if both inputs 1; OR outputs 1 if any input 1; NOT inverts input. Truth tables list all input-output combinations for each gate.

سوال 2: AND، OR، NOT گیٹس کی تعریف اور ٹروتھ ٹیبل دیں۔

جواب: AND: دونوں 1 ہوں تو 1؛ OR: کوئی بھی 1 ہو تو 1؛ NOT: الٹ دیتا ہے۔ ٹروتھ ٹیبل سے تمام ممکنہ جوابات ظاہر ہوتے ہیں۔

Q3: Simplify Boolean expression: (A·B)+(A·B').

Ans: Factor A: A·(B + B') = A·1 = A. So expression simplifies to A.

سوال 3: Boolean اظہار (A·B)+(A·B') کو سادہ کریں۔

جواب: A·(B+B')=A·1=A۔

Q4: Explain half adder and full adder.

Ans: Half adder adds two bits producing sum and carry (sum = A⊕B, carry = A·B). Full adder takes A, B, and carry-in producing sum and carry-out; full adders chain for multi-bit addition.

سوال 4: ہاف ایڈرز اور فل ایڈرز کی وضاحت کریں۔

جواب: ہاف ایڈر دو بٹس کا جمع کرتا ہے؛ فل ایڈر کارری-ان شامل کر کے تین بٹس کا جمع کرتا ہے۔

Q5: What is parity bit and how used for error detection?

Ans: Parity bit is added to data to make number of 1s even or odd. Receiver checks parity to detect single-bit errors. Simple detection but not error correction.

سوال 5: پیریٹی بٹ کیا ہے اور یہ ایرر ڈیٹیکشن میں کیسے استعمال ہوتا ہے؟

جواب: پیریٹی بٹ ڈیٹا میں شامل کیا جاتا ہے تاکہ 1s کی گنتی جفت یا طاق ہو؛ وصول کنندہ غلطی کا پتہ لگا سکتا ہے مگر درست نہیں کر سکتا۔

Q6: Convert decimal 45 to binary and hex.

Ans: 45₁₀ = 101101₂. Hex: group bits 0010 1101 → 2D₁₆.

جواب: 45 = 101101₂ = 2D₁₆۔

Q7: Describe combinational vs sequential circuits.

Ans: Combinational circuits' outputs depend only on current inputs (e.g., adders), sequential circuits depend on current inputs and past states (use memory elements like flip-flops), used in counters and registers.

سوال 7: کومبینیشنل اور سیکوینشل سرکٹس میں فرق بتائیں۔

جواب: کومبینیشنل صرف موجودہ ان پٹس پر منحصر ہوتا ہے؛ سیکوینشل ماضی کی حالت پر بھی منحصر ہوتا ہے اور میموری استعمال کرتا ہے۔

Q8: Explain truth table generation for expression (A + B)·C.

Ans: Compute A+B then AND with C for all 8 combos of A,B,C. The result is 1 only when C=1 and (A or B) =1.

سوال 8: اظہار (A + B)·C کے لیے ٹروتھ ٹیبل بنائیں۔

جواب: A OR B کا نتیجہ لے کر اسے C کے ساتھ AND کریں؛ نتیجہ تب 1 ہوگا جب C=1 اور (A یا B)=1 ہو۔

Q9: What is hexadecimal system and where used?

Ans: Hex is base-16 (0–9, A–F). Used in computing for compact representation of binary (e.g., memory addresses, color codes).

سوال 9: ہیکساڈیسمل سسٹم کیا ہے اور کہاں استعمال ہوتا ہے؟

جواب: ہیکسا بیس-16 ہے؛ کمپیوٹنگ میں بائنری کو مختصر انداز میں ظاہر کرنے کے لیے استعمال ہوتا ہے۔

Q10: Explain Boolean algebra laws: De Morgan's theorems.

Ans: De Morgan: (A·B)' = A' + B' and (A + B)' = A'·B'. Useful for simplifying and implementing logic circuits with NAND/NOR gates.

سوال 10: Boolean الجبرا کے De Morgan اصول بتائیں۔

جواب: (A·B)' = A' + B' اور (A + B)' = A'·B'۔ یہ سادہ کرنے اور سرکٹس میں ضرورت کے مطابق استعمال ہوتے ہیں۔

جواب: IoT آلات کو انٹرنیٹ سے جوڑتا ہے؛ اجزاء: سینسرز، ایکچیویٹر، کنیکٹیویٹی، ایج/گیٹ وے، کلاؤڈ۔

جواب: AR حقیقی دنیا پر ڈیجیٹل مواد اوورلے کرتا ہے؛ VR مکمل طور پر ورچوئل ماحول بناتا ہے۔

Q9: Describe practical use of IoT in smart agriculture.

7️⃣ Legal & Ethical Aspects Cyber Laws • Privacy • Ethical Hacking • Copyright
Topics:
  • Cybercrime types, privacy protection, ethical hacking, IP & copyright
Q1: Define cybercrime and list common types.

Ans: Cybercrime includes illegal activities like hacking, phishing, identity theft, malware distribution and online fraud. It harms users and organizations financially and reputationally.

سوال 1: سائبر کرائم کی تعریف کریں اور عام اقسام بتائیں۔

جواب: ہیکنگ، فشنگ، شناختی چوری، میلویئر، آن لائن فراڈ جیسی غیر قانونی سرگرمیاں سائبر کرائم کہلاتی ہیں۔

Q2: Explain digital privacy best practices.

Ans: Use strong passwords, two-factor authentication, limit data sharing, use encryption, update software and be cautious of suspicious links to protect personal data online.

سوال 2: ڈیجیٹل پرائیویسی کے بہترین طریقے بتائیں۔

جواب: مضبوط پاس ورڈ، 2FA، ڈیٹا شیئرنگ محدود کریں، انکرپشن استعمال کریں اور مشکوک لنکس سے ہوشیار رہیں۔

Q3: What is ethical hacking and how is it performed?

Ans: Ethical hacking is authorized penetration testing to find vulnerabilities. Steps: scope definition, reconnaissance, scanning, exploitation (in controlled way), reporting and remediation guidance.

سوال 3: ethical hacking کیا ہے اور اسے کیسے کیا جاتا ہے؟

جواب: یہ مجوزہ جانچ ہے جس کا مقصد کمزوریوں کو تلاش کر کے رپورٹ کرنا ہوتا ہے؛ اقدامات میں اسکوپ، نقشہ کشی، سکیننگ، کنٹرولڈ ایکسپلائیٹیشن اور رپورٹنگ شامل ہے۔

Q4: Describe copyright and software licensing basics.

Ans: Copyright grants creator exclusive rights; software licenses define permitted use. Types: proprietary (restricted use), permissive open-source (MIT), copyleft (GPL) which requires derivative works to be open.

سوال 4: کاپی رائٹ اور سافٹ ویئر لائسنسنگ کی بنیادی باتیں بتائیں۔

جواب: کاپی رائٹ تخلیق کار کو حقوق دیتا ہے؛ لائسنس استعمال کی شرائط بتاتے ہیں؛ اقسام میں پروپرائٹری اور مختلف اوپن سورس لائسنس شامل ہیں۔

Q5: What steps should organizations take for cybersecurity?

Ans: Implement firewalls, keep systems patched, use encryption, perform regular backups, conduct employee training and vulnerability assessments, and prepare an incident response plan.

سوال 5: تنظیموں کو سائبر سیکیورٹی کے لیے کیا اقدامات کرنے چاہئیں؟

جواب: فائر والز، پیچز، انکرپشن، بیک اپ، ملازمین کی تربیت، وَلنرایبیلٹی اسیسمنٹس اور انسڈینٹ رسپانس پلان تیار کریں۔

Q6: Explain data breach and response best practices.

Ans: Data breach is unauthorized access to data. Response: contain breach, assess impact, notify affected parties, patch vulnerabilities, conduct forensic analysis and improve security posture.

سوال 6: ڈیٹا بریچ کیا ہے اور اس کے ردِعمل کی بہترین مشقیں کیا ہیں؟

جواب: ڈیٹا بریچ میں غیر مجاز رسائی شامل ہے؛ فوری طور پر کنٹین کریں، متاثرین کو مطلع کریں، خامیوں کو درست کریں اور فورینسک کریں۔

Q7: What is intellectual property (IP) and its types?

Ans: IP covers creations: copyrights (creative works), patents (inventions), trademarks (brands) and trade secrets. IP gives legal protection and economic rights to creators.

سوال 7: intellectual property کیا ہے اور اقسام بتائیں؟

جواب: IP میں کاپی رائٹ، پیٹنٹ، ٹریڈ مارک اور ٹریڈ سیکریٹس شامل ہیں جو تخلیق کار کو قانونی حقوق فراہم کرتے ہیں۔

Q8: Discuss privacy laws and user rights (conceptual).

Ans: Privacy laws regulate data collection, require consent, allow access/deletion rights and mandate breach notifications. They aim to protect individuals’ personal data and privacy.

سوال 8: پرائیویسی قوانین اور صارف کے حقوق پر مختصر بیان کریں۔

جواب: قوانین ڈیٹا جمع کرنے کو منظم کرتے ہیں، رضا مندی درکار کرتے ہیں اور رسائی/حذف کے حقوق مہیا کرتے ہیں۔

Q9: What is responsible disclosure?

Ans: Responsible disclosure means privately notifying vendors about vulnerabilities, allowing time to fix before public disclosure, to avoid exploitation and enable patching.

سوال 9: responsible disclosure کیا ہے؟

جواب: ذمہ دارانہ انکشاف کا مطلب ہے کہ خامیوں کو پوشیدہ طور پر اطلاع دیں تاکہ وینڈر پریکٹس کر کے انہیں درست کر سکے اس سے پہلے کہ عوامی طور پر ظاہر کیا جائے۔

Q10: Explain ethics of data collection and consent.

Ans: Collect only necessary data with informed consent, be transparent about usage, secure storage and allow users to opt-out. Ethical collection respects privacy and builds trust.

8️⃣ Online Research & Digital Literacy Searching • Plagiarism • Source Evaluation
Topics:
  • Advanced search operators, evaluating credibility, citing sources, digital footprint
Q1: What is plagiarism and how to avoid it?

Ans: Plagiarism is using others' work without credit. Avoid by citing sources, paraphrasing properly and using plagiarism checkers and reference managers.

سوال 1: چوریِ علمی (plagiarism) کیا ہے اور اس سے کیسے بچیں؟

جواب: دوسرے کام کو بغیر حوالہ کے پیش کرنا plagiarism کہلاتا ہے؛ حوالہ دیں اور مناسب پیرہ گراف استعمال کریں۔

Q2: How to evaluate online sources for credibility?

Ans: Check author credentials, publication date, references, domain type (.edu/.gov), cross-check facts and spot bias. Use reputable journals and primary sources where possible.

سوال 2: آن لائن ذرائع کی قابلِ اعتباریت کیسے جانچیں؟

جواب: مصنف کی اہلیت، تاریخ، حوالہ جات، ڈومین اور مختلف ذرائع سے تصدیق کریں۔

Q3: Explain advanced search operators (quotes, site:, filetype:).

Ans: Use quotes for exact phrases, site: to limit domain, filetype:pdf to find PDFs, minus (-) to exclude terms and OR for alternatives. These refine results and save time.

سوال 3: جدید سرچ آپریٹرز کی وضاحت کریں۔

جواب: "quotes"، site:، filetype: وغیرہ مخصوص تلاش کے لیے استعمال ہوتے ہیں۔

Q4: What is digital footprint and its implications?

Ans: Digital footprint is the trail of online activities. It affects privacy and reputation and may be reviewed by employers or institutions; manage it via privacy settings and mindful posting.

سوال 4: ڈیجیٹل فٹ پرنٹ کیا ہے اور اس کے نتائج کیا ہیں؟

جواب: آن لائن سرگرمیوں کا ریکارڈ ڈیجیٹل فٹ پرنٹ ہے؛ یہ رازداری اور شہرت پر اثر انداز ہو سکتا ہے۔

Q5: How to cite online sources in academic work (basic)?

Ans: Include author, title, website, publication date and access date/URL. Follow citation style (APA/MLA/IEEE) required by institution.

سوال 5: آن لائن ذرائع کو اکیڈمک ورک میں کیسے حوالہ دیں؟

جواب: مصنف، عنوان، سائٹ، تاریخ اور URL شامل کریں اور ضروری حوالہ اسٹائل استعمال کریں۔

Q6: What tools help with referencing and plagiarism detection?

Ans: Tools: Zotero/Mendeley for refs, Turnitin/Grammarly for plagiarism checks, Google Scholar for literature search.

سوال 6: حوالہ جات اور plagiarism چیک کے لیے کون سے ٹولز مدد دیتے ہیں؟

جواب: Zotero، Mendeley، Turnitin، Grammarly اور Google Scholar عام ٹولز ہیں۔

Q7: Describe fact-checking steps.

Ans: Verify source, cross-check claims with reputable outlets, check dates, inspect images for manipulation, consult original documents or data where possible.

سوال 7: fact-checking کے مراحل بتائیں۔

جواب: ماخذ کی جانچ، دوسری معتبر خبروں سے تصدیق، تاریخیں چیک کرنا اور اصل دستاویزات دیکھنا شامل ہے۔

Q8: What is primary vs secondary source?

Ans: Primary sources are original data (experiments, interviews), secondary analyze or summarize primary sources (reviews, articles). Prefer primary for original evidence.

سوال 8: primary اور secondary ذرائع میں فرق بتائیں۔

جواب: پرائمری اصل مواد ہوتا ہے؛ سیکنڈری ان کا تجزیہ یا خلاصہ ہوتا ہے۔

Q9: How to ethically use images from the web?

Ans: Check licenses (Creative Commons), attribute authors when required, use public domain or royalty-free images, and avoid copyrighted material without permission.

سوال 9: ویب سے تصاویر کو اخلاقی طور پر کیسے استعمال کریں؟

جواب: لائسنس چیک کریں، مصنف کا حوالہ دیں، پبلک ڈومین یا رائلٹی فری تصاویر استعمال کریں۔

Q10: How to build a concise literature review?

Ans: Define scope, search widely, summarize key findings, identify gaps, organize thematically, and cite sources properly to justify your research.

سوال 10: ایک مختصر literature review کیسے بنائیں؟

جواب: دائرہ کار طے کریں، وسیع تلاش کریں، کلیدی نتائج خلاصہ کریں، خالی جگہیں دکھائیں اور مناسب حوالہ دیں۔
